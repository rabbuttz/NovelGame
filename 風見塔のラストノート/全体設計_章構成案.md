## 章ごとの詳細

## 第1章「転入初日、封鎖塔と仮チーム」
- **章の目的**
  - 主人公の転入導入、風見塔との遭遇、主要人物の第一印象提示。
  - 「再公開企画」が無茶であることを最初に刻む。
- **主に掘るキャラ**
  - きりゅう / あむ / さわでん（導入トリオ）
  - 補助: アガサ、すずどら（企画の温度差提示）
- **入れるべきイベント**
  - 校内迷子→風見塔前で初遭遇
  - 仮の作業打ち合わせ（雑務係任命）
  - 「想いを伝えると結ばれる塔」の噂初出
  - 生徒会・実行委員・現場側の温度差を会話で提示
- **必須の感情変化**
  - 主人公: 「部外者」→「巻き込まれた当事者」
  - きりゅう: 警戒→最低限の信頼
  - あむ: 興味本位→仲間意識の芽
- **分岐に使う変数やフラグ候補**
  - `flags.metKazamiTower`
  - `flags.acceptedErrandRole`
  - `flags.heardTowerRumor`
  - `affection.kiryu / affection.amu / affection.agasa / affection.sawaden`
- **章末の引き**
  - 風見塔内部の古い掲示に「最終公開記録」の文言。誰が最後に使ったのか不明。

## 第2章「役割決めと最初の摩擦」
- **章の目的**
  - チーム体制確定、全員の役割を見せる。
  - 各キャラが“必要な戦力”である状態を作る。
- **主に掘るキャラ**
  - うきわ / ぐら / 餅虎 / ねむのみ / みもち
- **入れるべきイベント**
  - 役割会議（安全管理・修繕・広報・来場導線）
  - うきわの段取りミスとフォロー
  - ぐらの設備点検（静かな有能描写）
  - 餅虎の宣伝案（派手だが賛否）
  - みもちのアーカイブ整理提案（過去資料の見える化）
  - ねむのみの休憩導線設計（チームの緩衝）
- **必須の感情変化**
  - 主人公: 「誰を手伝うべきか迷う」→「横断役を引き受ける」
  - チーム: まとまりの無さ→最低限の協業
- **分岐に使う変数やフラグ候補**
  - `flags.helpedUkiwaFirst`
  - `flags.backedMochitoraPlan`
  - `flags.acceptedMimochiArchivePlan`
  - `flags.joinedGuraInspection`
  - `flags.restWithNemunomi`
  - `workload.supportCount`
- **章末の引き**
  - 点検中、封鎖区画に「旧公開中止届」の控えを発見。提出者名が読めない。

## 第3章「許可取りの表と裏」
- **章の目的**
  - 学内政治（生徒会・教員・実行委員）の対立軸を立てる。
  - アガサとすずどらの価値観差を提示。
- **主に掘るキャラ**
  - アガサ / すずどら / きりゅう
  - 補助: ロクロ（非攻略の障害兼評価者）
- **入れるべきイベント**
  - 公開申請ドラフト作成
  - アガサの根回し成功と、その副作用（反感）
  - すずどらの正論で企画停止寸前
  - きりゅうの安全基準説明（感情ではなく責任）
- **必須の感情変化**
  - 主人公: 「通せば勝ち」→「通し方にも責任がある」
  - アガサ: 余裕→焦りの兆候
  - すずどら: 冷淡に見える→本気で事故を恐れていると判明
- **分岐に使う変数やフラグ候補**
  - `flags.choseFormalApprovalPath`
  - `flags.trustedAgasaNegotiation`
  - `flags.facedSuzudoraDirectly`
  - `flags.readSafetyGuideline`
  - `affection.agasa / affection.suzudora / affection.kiryu`
- **章末の引き**
  - 許可は「条件付き仮承認」。ただし一つでも基準未達なら即時中止。

## 第4章「噂の正体と失われたノート」
- **章の目的**
  - 物語中核ミステリ（ラストノート）を表に出す。
  - 恋愛噂と事故噂の二重構造を明確化。
- **主に掘るキャラ**
  - さわでん / ぐら / ねむのみ
- **入れるべきイベント**
  - さわでんが古い鍵束とノート断片を拾う
  - ぐらが塔内機構から「隠し保管箱」を推定
  - ねむのみが過去に傷ついた生徒の話を聞き出す
  - ノートの一部復元（誰かの未送信メッセージ）
- **必須の感情変化**
  - 主人公: 好奇心→「掘るほど誰かを傷つけるかもしれない」葛藤
  - さわでん: 無邪気→責任の自覚
- **分岐に使う変数やフラグ候補**
  - `flags.foundOldKey`
  - `flags.collectedNoteFragments >= 3`
  - `flags.sharedTruthWithTeam`
  - `flags.keptNoteSecret`
  - `lore.noteProgress`
- **章末の引き**
  - ノートの最後のページだけが欠落。ページの切り取り跡が新しい。

## 第5章「準備加速、関係深化、見え始めるほころび」
- **章の目的**
  - 日常イベント群で好感度を厚く積み上げる中盤の核。
  - 後半崩壊の前に“楽しい成功体験”を作る。
- **主に掘るキャラ**
  - 全攻略対象を均等に（重点: あむ / うきわ / 餅虎 / みもち / どり）
- **入れるべきイベント**
  - 放課後作業（装飾・導線・配布物）
  - 広報本番（餅虎主導の校内放送・SNS相当掲示）
  - みもち主導の「風見塔メモリー展示」準備（写真・記録抜粋）
  - どりの静かな実務支援（不足物資の補完）
  - 小成功の積み重ね（公開前プレ動線テスト成功）
- **必須の感情変化**
  - 主人公: 「やれるかも」実感
  - チーム: バラバラ→“居場所”化
  - 各キャラ: 主人公への私的な期待が強まる
- **分岐に使う変数やフラグ候補**
  - `affection.*`（全員の閾値チェック）
  - `flags.stayedLateWithKiryu`
  - `flags.chasedAmuAfterFailure`
  - `flags.encouragedUkiwa`
  - `flags.reliedOnDori`
  - `flags.sawMochitoraAnxiety`
  - `flags.helpedMimochiCuration`
- **章末の引き**
  - 公開2日前、匿名投書で「風見塔再公開差し止め」審議が緊急招集。

## 第6章「公開中止危機とチーム崩壊寸前」
- **章の目的**
  - 共通ルート最大ストレス。対立と本音の噴出。
  - ここで“誰の手を取るか”が分岐の本体になる。
- **主に掘るキャラ**
  - すずどら / きりゅう / アガサ（責任ライン）
  - あむ / うきわ（感情ライン）
- **入れるべきイベント**
  - 公開中止の可能性通告
  - 「安全優先派 vs 実行強行派」正面衝突
  - アガサの根回しが逆風になる
  - あむ・うきわの失敗連鎖
  - 主人公の個別フォロー選択（複数回）
- **必須の感情変化**
  - 主人公: 調整役の限界→それでも繋ぐ決意
  - チーム: 相互不信→最低限の再接続
- **分岐に使う変数やフラグ候補**
  - `flags.supportedSafetyFirst`
  - `flags.supportedFestivalFirst`
  - `flags.reconciledKeyPair`（例: kiryu-amu, suzudora-agasa）
  - `trust.teamCohesion`
  - `pressure.crisisMeter`
- **章末の引き**
  - 欠落していた「ラストノート最終頁」が提出資料に紛れ込み、過去の誤解を覆す可能性が示される。

## 第7章「前夜、最後の選択」
- **章の目的**
  - 分岐直前の情緒最大化。
  - 各キャラの“恋愛としての本音”を初めて明確にする。
- **主に掘るキャラ**
  - 全攻略対象（2人ずつ対比配置）
- **入れるべきイベント**
  - 前夜の塔内最終確認
  - 欠落ページの真相共有/秘匿の最終選択
  - 個別深夜会話（キャラ別優先発生）
  - 主人公の「誰を信じ、誰に頼るか」決断
- **必須の感情変化**
  - 主人公: みんなを守る→“一人を特別に選ぶ”怖さの自覚
  - 各キャラ: 仲間としての距離→恋愛として踏み込む覚悟
- **分岐に使う変数やフラグ候補**
  - `routeLock.<charId>`
  - `flags.revealedLastPageTruth`
  - `flags.promisedToMeet<CharId>`
  - `affection.<charId> >= threshold`
- **章末の引き**
  - 星潮祭当日朝、「最終公開可否は本日正午決裁」と告げられる。

## 第8章「星潮祭、風見塔の再点灯」
- **章の目的**
  - 共通ルート決着と個別導入フックの同時達成。
  - 成功/綱渡り成功の2系統で個別へ接続可能にする。
- **主に掘るキャラ**
  - 全体群像 + ルート確定上位キャラ
- **入れるべきイベント**
  - 正午決裁→公開可否決定
  - 一日限定公開スタート
  - 来場者反応（噂の再燃と鎮静）
  - 夜、塔最上階で「ラストノート」を巡る最終対話
  - 主人公が特定キャラと二人になる導線
- **必須の感情変化**
  - 主人公: 企画達成感→個人的な恋の自覚
  - チーム: 企画仲間→それぞれの未来へ
- **分岐に使う変数やフラグ候補**
  - `flags.commonRouteCleared`
  - `route.target`
  - `ending.commonOutcome`（perfect / narrow）
- **章末の引き**
  - 「この夜を境に、もう“ただの仲間”には戻れない」で個別ルート開始。

## 分岐判定インタールード「ラストノートの宛先」
- **役割**
  - 共通ルート最終フラグ集計とルート確定演出。
- **実装意図（YAML化前提の設計段階）**
  - 好感度トップのみで即決せず、必須イベント達成を重視。
  - 未達成時はノーマル保留または再選択導線を用意。
- **判定優先度案**
  1. 必須フラグ達成
  2. 章6〜7の核心選択一致
  3. 好感度
  4. タイブレーク用の直近選択

## キャラ別ルート一覧（導入条件）

> 注: ここでは**既存実装で確認できる攻略対象ID（きりゅう / アガサ / あむ / うきわ / ぐら / すずどら / ねむのみ / 餅虎）**に加えて、追加連絡のあった**みもち**を導入対象として条件化。さわでん・どりは本企画では共通重要キャラとして扱い、個別導入は将来拡張枠とする。

### きりゅうルート
- 最低条件
  - `affection.kiryu >= 既定閾値`
  - `flags.stayedLateWithKiryu == true`
  - `flags.supportedSafetyFirst == true`（または中立でも安全基準を守る選択）
- 導入トリガー
  - 星潮祭終了後「片付けを最後まで手伝う」を選択。

### アガサルート
- 最低条件
  - `affection.agasa >= 既定閾値`
  - `flags.trustedAgasaNegotiation == true`
  - `flags.revealedLastPageTruth`の扱いでアガサの信頼方針と一致
- 導入トリガー
  - 夜の塔で「噂の真相より、君の本音を聞きたい」を選択。

### あむルート
- 最低条件
  - `affection.amu >= 既定閾値`
  - `flags.chasedAmuAfterFailure == true`
  - 第6章であむの失敗を責めず、再起を促す選択達成
- 導入トリガー
  - 前夜または当夜に「走って追いかける」系選択を選ぶ。

### うきわルート
- 最低条件
  - `affection.ukiwa >= 既定閾値`
  - `flags.helpedUkiwaFirst == true` または `flags.encouragedUkiwa == true`
  - 締切事故イベントで見捨てない選択
- 導入トリガー
  - 祭り後「一緒にやり残しを片づける」を選択。

### ぐらルート
- 最低条件
  - `affection.gura >= 既定閾値`
  - `flags.joinedGuraInspection == true`
  - 夜間作業イベント複数回参加
- 導入トリガー
  - 塔の機材停止後「二人で最終点検する」を選択。

### すずどらルート
- 最低条件
  - `affection.suzudora >= 高め閾値`
  - `flags.facedSuzudoraDirectly == true`
  - 第6章で対立から逃げない選択を連続達成
- 導入トリガー
  - 決裁直前「反発されても向き合う」を選択。

### ねむのみルート
- 最低条件
  - `affection.nemunomi >= 既定閾値`
  - `flags.restWithNemunomi == true`
  - 看病/相談イベントで本音を返す選択達成
- 導入トリガー
  - 祭り後の保健室シーンで「今度はあなたの話を聞きたい」を選択。

### 餅虎ルート
- 最低条件
  - `affection.mochitora >= 既定閾値`
  - `flags.backedMochitoraPlan == true`
  - `flags.sawMochitoraAnxiety == true`
- 導入トリガー
  - ステージ撤収後「からかいの裏にある不安を受け止める」を選択。

### みもちルート
- 最低条件
  - `affection.mimochi >= 既定閾値`
  - `flags.acceptedMimochiArchivePlan == true`
  - `flags.helpedMimochiCuration == true`
  - 第7章で「ノートの記録を未来に残す」方針選択を達成
- 導入トリガー
  - 星潮祭終了後、展示撤収シーンで「最後の1ページを一緒に綴じる」を選択。

## 実装前メモ（次段階への受け渡し）
- 次工程では本設計を章ごとに分割し、**1章ずつ YAML 化**する。
- 章ごとの YAML 作成時は、既存の表情運用（`default/smile/cry/troubled` 優先）を厳守する。
- 分岐判定は「好感度だけでなく必須イベント達成」を前提にする。
- さわでん・どり個別は、共通ルート完走後の追加DLC章またはIFルート枠として設計可能。
